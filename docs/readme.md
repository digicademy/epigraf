# Epigraf Developer Documentation

Epigraf's developer documentation is generated by [Jekyll](https://jekyllrb.com/) from
the Markdown files in this directory
Jekyll processes all files that contain a [front matter section](https://jekyllrb.com/docs/front-matter/).

Make sure the pages of the user documentation render nicely in the Epigraf help section.
While writing, you can inspect the rendered pages in your local Epigraf instance.

When working on the developer documentation, you can use a jekyll container to display the pages.

## Conventions

- Wherever possble, integrate new pages into the existing structure.
- Make sure the documentation is comprehensible regardless of the user's domain
  or specific databases. For examples, you can refer to the movies database,
  but you need to show how to set up the movies database in the first place.

## Links
- Never use absolute URLs in the documentation.
  Always use relative links, e.g. `/user/administration/databases`.
  Omit trailing slashes and the md extension. For links to index pages,
  link to the folder, e.g. `/user/administration`.
  The `/user/` path element is necessary for the Jekyll generated documentation.
  It is automatically replaced by `/help/` when the user documentation is rendered in the Epigraf help section.
- When linking to other pages, avoid phrases such as "look here" or "see xyz".
  Instead, write your explanation and link relevant words to the respective page.

## Images
- Include screenshots but keep the number of screenshots to a minimum.
- Make sure the screenshots use the default theme and keep a consistent
  visual style. For example, explanation boxes and arrows use red lines,
  with the always same line width. Don't use round corners.
- Place screenshots in the `assets` folder and link to them using relative URLs.
  Include an image description.
  For image paths, the `/user/` path element is necessary for the Jekyll generated documentation.
  It is automatically replaced by `/help/` when the user documentation is rendered in the Epigraf help section.
  Example: `![Image showing the RAM](/epigraf/user/assets/ram_simple.png)`.
- To adjust the image sizes, use HTML markup.
  Add a `<figure>` element around the `<img>` element.
  Then use the `style` attribute of the figure element to set the width and optionally a bottom margin,
  e.g. `<figure style="width: 400px;margin-bottom:2em;">`.
  Don't forget the alt attribute of the image element.
- For assets, use file names that are descriptive of the content (avoid abbreveations such as `gs_artv-cols`)
  and that indicate on which pages or for what purposes the image is used.
  For example, all images related to the RAM should start with `ram_`.
- You can use mermaid to create diagrams. Store the mermaid code in a separate mmd file in the `assets` folder.
  Save the rendered diagram as png and use the png file in the documentation.
